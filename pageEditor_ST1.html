2. Technical Description of Implemented Pages

Global CSS & Template Structure

I designed a dark-themed "cyber-glass" aesthetic using CSS variables (:root) to maintain strict color consistency across the team's files. The shared template relies on semantic HTML5 tags (<header>, <nav>, <main>, <footer>). For layout control, I heavily utilized CSS Flexbox to ensure the navigation bar and footer elements respond cleanly to viewport changes. To protect the integrity of the design, I inserted explicit "DO NOT MODIFY" comments bounding the template areas.

Home Page

Live Page Validation

The Home Page leverages CSS Grid and Flexbox for its layout. The core mission is displayed using a card-based UI. I achieved the "frosted glass" look (Glassmorphism) by combining background: rgba(...) with the backdrop-filter: blur() property. The page also includes an advanced interactive Bento Grid with 3D magnetic mouse-tracking and real-time scroll parallax.

Gallery Page

Live Page Validation

The Gallery employs a responsive CSS Grid (grid-template-columns: repeat(...)) to arrange the thumbnails. The core functionality is driven by JavaScript. When a thumbnail is clicked, the openModal() function extracts data from the DOM and injects it into a centralized modal overlay. I also implemented JavaScript customization controls that modify the DOM's inline styles to toggle Light/Dark themes and dynamically adjust the fontSize without reloading.

Content Page (Institutional Corruption)

This page demonstrates structured content hierarchy using semantic tags. For navigation, I implemented an internal Table of Contents using HTML anchor links. Furthermore, I engineered an advanced scroll-linked engine that injects an automated progress bar and drives visual parallax effects on the article images to create a premium editorial reading experience.

3. Challenges & Lessons Learned

One major technical challenge was "scroll thrashing" on the shared navigation header. I initially used a basic JavaScript scroll listener to shrink the header when the user scrolled past 50px. However, if the user scrolled very slowly at that exact threshold, the document height change would cause the header to infinitely loop between its expanded and shrunk states. I resolved this by adding hysteresis (a buffer) to the logic—the header shrinks at >80px, but doesn't expand again until the user scrolls back above <20px. This taught me the importance of edge-case testing in UI event listeners.

4. Accessibility Considerations

Ensuring our website is accessible to all users was a primary focus during development. I implemented several techniques to improve usability and inclusivity:

  • Semantic HTML: I used appropriate HTML5 structural tags (<header>, <nav>, <main>, <section>, <article>) to ensure screen readers can accurately interpret the page layout and navigation flow.
  • Alternative Text: All images across the Home, Gallery, and Content pages include descriptive alt attributes, ensuring visually impaired users can understand the visual context and purpose of the images.
  • Color Contrast: The "cyber-glass" theme uses a dark background (#05070a) with light silver/muted text (#e2e8f0 and #94a3b8), which provides high contrast to reduce eye strain and improve readability for all users.
  • Keyboard Navigation: Interactive elements, including the navigation links and "Go to Top" buttons, are structured to be fully accessible via keyboard tabbing.
