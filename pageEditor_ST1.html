2. Technical Description of Implemented Pages

Global CSS & Template Structure

I designed a dark-themed "cyber-glass" aesthetic using CSS variables (:root) to maintain strict color consistency across the team's files. The shared template relies on semantic HTML5 tags (<header>, <nav>, <main>, <footer>). For layout control, I heavily utilized CSS Flexbox to ensure the navigation bar and footer elements respond cleanly to viewport changes. To protect the integrity of the design, I inserted explicit "DO NOT MODIFY" comments bounding the template areas.

Home Page

View Live Home Page → | View Validation

The Home Page leverages CSS Grid and Flexbox for its layout. The core mission is displayed using a card-based UI. I achieved the "frosted glass" look (Glassmorphism) by combining background: rgba(...) with the backdrop-filter: blur() property. The page also includes embedded CSS to style the interactive "Explore Our Research" tiles, featuring CSS transitions (transform: scale()) that provide immediate visual feedback upon user hover.

Gallery Page

View Live Gallery Page → | View Validation

The Gallery employs a responsive CSS Grid (grid-template-columns: repeat(auto-fit, minmax(...))) to arrange the thumbnails. The core functionality is driven by JavaScript. When a thumbnail is clicked, the openModal() function extracts data (image source, alt text, and hidden paragraph descriptions) from the DOM and injects it into a centralized modal overlay. I also implemented JavaScript customization controls that modify the DOM's inline styles to toggle Light/Dark themes and dynamically adjust the fontSize in REM units without reloading the page.

Content Page (Institutional Corruption)

View Live Content Page → | View Validation

This page demonstrates structured content hierarchy using <h1> down to <h3> tags. For navigation, I implemented an internal Table of Contents using HTML anchor links (href="#id"). Furthermore, I engineered a "Go to Top" button entirely out of HTML/CSS by setting its container to position: fixed; bottom: 30px;, ensuring it remains accessible regardless of scroll depth.

3. Challenges & Lessons Learned

One major technical challenge was "scroll thrashing" on the shared navigation header. I initially used a basic JavaScript scroll listener to shrink the header when the user scrolled past 50px. However, if the user scrolled very slowly at that exact threshold, the document height change would cause the header to infinitely loop between its expanded and shrunk states. I resolved this by adding hysteresis (a buffer) to the logic—the header shrinks at >80px, but doesn't expand again until the user scrolls back above <20px. This taught me the importance of edge-case testing in UI event listeners.

4. Accessibility Considerations

Accessibility was prioritized across all my pages to ensure inclusivity. Every image utilized in the Home, Gallery, and Content pages includes descriptive alt text, enabling screen readers to interpret visual information. Furthermore, I enforced strict contrast ratios in the global CSS; text elements use high-contrast whites and light greys against the deep dark background to assist users with visual impairments. Semantic tags were used strictly for their intended structural purposes, avoiding the misuse of <div> tags where <article> or <section> was more appropriate.

5. JANET Compliance

The website strictly adheres to the JANET Acceptable Use Policy. All content focuses academically and constructively on SDG 16 (Peace, Justice, and Strong Institutions) without engaging in defamatory, offensive, or politically partisan material. The site does not host commercial advertising, nor does it consume excessive network bandwidth (all images are highly compressed and appropriately sized). Furthermore, all external visual assets were sourced from royalty-free platforms (Unsplash) and are fully referenced to respect intellectual property rights.
